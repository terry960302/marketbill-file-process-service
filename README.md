## 성능 개선

###  접근 방식 
- PDF 각 컴포넌트 생성시 동기적으로 이뤄지니 컴포넌트 생성을 비동기에 병렬로 변경(asyncio.gather)
- 연산 작업이 대부분이므로 멀티프로세싱으로 요청 처리

### 결과
- 컴포넌트 병렬 처리
  - 기존에
- 멀티프로세싱 요청 처리
  : 단건에선 개선이 없으나 여러 요청에서 기하급수적인 개선이 이뤄짐.
  - 1개 : 2.3s
  - 50개 : 4.9s
  - 100개 : 8.9s
